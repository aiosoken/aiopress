# AIOプレス 技術スタック選定書 (ハッカソン要件対応)

## 1. はじめに

本ドキュメントは、AIOプレス開発プロジェクトにおいて、提示されたハッカソンの技術要件[1]に基づき、採用するGoogle Cloudのプロダクトを選定し、その理由を明確化することを目的とします。

プロジェクトの目的は、AIとデザインシステムを融合させた「Brand Ops基盤」を構築することです。この目的達成のために、機能性、開発効率、スケーラビリティ、そしてハッカソンという短期間での開発における実現可能性を総合的に評価し、最適な技術スタックを選択しました。

## 2. ハッカソン必須要件の確認

- **アプリケーション実行プロダクト**: 以下のいずれかを1つ以上利用すること。
  - App Engine, Google Compute Engine, Google Kubernetes Engine (GKE), **Cloud Run**, **Cloud Functions**, Cloud TPU / GPU
- **AI技術**: 以下のいずれかを1つ以上利用すること。
  - **Vertex AI**, **Gemini API**, Gemma, Imagen, Agent Builder, ADK, Speech-to-Text / Text-to-Speech API, **Vision AI API**, **Natural Language AI API**, Translation AI API

## 3. 選定技術スタックとアーキテクチャ

上記の要件と、本プロジェクトのシステム設計[2]を考慮し、以下の技術スタックを選定しました。

| 分類 | 選定プロダクト | 選定理由 |
| :--- | :--- | :--- |
| **フロントエンド実行** | **Cloud Run** | ・Next.jsで構築されたWebアプリケーションをコンテナとしてデプロイするのに最適です。<br>・サーバーレスであり、トラフィックに応じて自動でスケールし、アクセスがない場合はゼロになるため、コスト効率に優れています。<br>・GCEやGKEのようなインフラ管理が不要で、開発者はアプリケーションコードに集中できます。 |
| **バックエンド実行** | **Cloud Functions** | ・AIによる分析や評価といった、イベントドリブンで実行されるバックエンド処理に最適です。<br>・特定のHTTPリクエストに応じてAIチェック機能を実行したり、ファイルストレージへのアップロードをトリガーに資産分析を実行したりするのに適しています。<br>・Cloud Run同様、サーバーレスで管理が容易です。 |
| **AI/ML基盤** | **Vertex AI** | ・本プロジェクトで利用する複数のAIモデル（Gemini, Vision AIなど）を統合的に管理・利用するための最適なプラットフォームです。<br>・認証情報やAPIエンドポイントを一元管理でき、開発体験が向上します。<br>・将来的にカスタムモデルをトレーニングするなど、機能拡張の際にも中心的な役割を果たします。 |
| **AI: 画像・テキスト分析** | **Vision AI API** | ・アップロードされたクリエイティブ画像から、使用されている色、ロゴ、テキスト（OCR）などを高精度に抽出する上で必須の機能です。<br>・AIチェック機能の根幹をなす「現状分析」の役割を担います。 |
| **AI: テキスト分析** | **Natural Language AI API** | ・Vision AIで抽出したテキストや、ドキュメント資産のテキストコンテンツを分析し、トーン＆マナーやエンティティを抽出するために利用します。<br>・ブランドの言語的側面を構造化する上で重要な役割を果たします。 |
| **AI: 総合評価・推論** | **Gemini API** | ・Vision AIとNatural Language AIによる分析結果と、Firestoreに保存されたブランドガイドラインやデザイントークンを総合的に比較・評価し、「ブランド準拠スコア」を算出するために利用します。<br>・高度な推論能力により、単なるルール比較に留まらない、「なぜ準拠していないのか」「どうすれば改善できるか」といった具体的な修正提案の生成が可能です。 |

### アーキテクチャへの反映

この技術選定は、システム設計書v2[2]で定義したアーキテクチャと完全に一致しています。フロントエンドは**Cloud Run**上で稼働し、AIを駆使した重い処理は**Cloud Functions**経由で**Vertex AI**上の各種AIモデル（**Gemini, Vision AI**など）を呼び出す構成となります。これにより、ハッカソンの要件を満たしつつ、モダンで効率的なWebアプリケーションを構築します。

## 4. 選定しなかった技術とその理由

| 分類 | 選定しなかったプロダクト | 理由 |
| :--- | :--- | :--- |
| **実行環境** | GCE, GKE | ・仮想マシンやコンテナオーケストレーションの管理は、本プロジェクトのスコープとハッカソンの期間に対して過剰な複雑さを持ち込みます。<br>・サーバーレスアプローチ（Cloud Run, Cloud Functions）の方が、開発効率とコスト効率の観点から優れています。 |
| **AI技術** | Imagen, Speech-to-Text等 | ・現時点の要件定義では、能動的な画像「生成」や音声認識は主要機能に含まれていません。<br>・プロジェクトのコア価値である「既存クリエイティブの評価と改善」に集中するため、スコープ外としました。将来的な機能拡張の候補とはなり得ます。 |

## 5. 結論

**Cloud Run**, **Cloud Functions**, **Vertex AI**, **Gemini API**, **Vision AI API** を組み合わせることで、ハッカソンの必須要件をすべて満たし、かつ本プロジェクトが目指す「Brand Ops基盤」のコア機能を、堅牢かつスケーラブルな形で実現することが可能です。この技術選定に基づき、実装フェーズへと進みます。
