# AIOプレス 要件定義書 v3 (クリエイティブ生成特化)

## 1. 概要

### 1.1. プロジェクトの目的

本プロジェクトは、AIによるクリエイティブ評価（Check機能）から、**AIによるクリエイティブ生成**へとコア機能を転換し、「AIOプレス」をブランドの一貫性を担保するコンテンツ生成プラットフォームとして再定義することを目的とする。ユーザーが定義した「ブランドDNA」に基づき、AIがキャッチコピー、SNS投稿、ブログ記事、バナー画像などの多様なクリエイティブを自動生成する仕組みを構築する。

### 1.2. コンセプト：AI時代のブランドパートナー

AIの「アイデア出し」「下書き作成」という強みを最大限に活かし、広報担当者のクリエイティブ業務を劇的に効率化する。単なるコンテンツ生成ツールではなく、入力されたブランドDNAを常に参照することで、**誰が使ってもブランドらしさが保たれる**「戦略的パートナー」となることを目指す。

### 1.3. スコープ

- **IN**:
    - ブランドDNA（理念、価値観、トーン＆マナー、デザイン原則）の管理機能。
    - AIによる各種クリエイティブ（テキスト、画像）の生成機能。
    - 生成されたクリエイティブの編集、管理、エクスポート機能。
    - ブランドアセット（ロゴ、参考資料）の管理機能。
- **OUT**:
    - AIによるクリエイティブ評価（Check機能）。これは将来的な拡張機能として位置づける。

## 2. 機能要件

### 2.1. コア機能：ブランドDNA × AIクリエイティブ生成

| 優先度 | 機能分類 | 機能名 | 機能概要 |
| :--- | :--- | :--- | :--- |
| **最高** | **ブランドDNA管理** | **ブランドDNA設定** | ブランドの核となる情報（ミッション、ビジョン、提供価値、ターゲット層、トーン＆マナー）をテキストで入力・管理できる。 |
| **最高** | | **デザインシステム設定** | ブランドの視覚的原則（プライマリカラー、セカンダリカラー、フォント、ロゴ）を設定・管理できる。 |
| **最高** | **AIクリエイティブ生成** | **テキスト生成** | ブランドDNAとユーザーの指示（目的、キーワード等）に基づき、AI（Gemini API）がキャッチコピー、SNS投稿文、ブログ記事などのテキストコンテンツを複数パターン生成する。 |
| **高** | | **画像生成** | ブランドDNA（特にデザインシステム）とユーザーの指示に基づき、AI（Imagen API）がSNS投稿用画像や広告バナーなどの画像を生成する。 |
| **高** | **クリエイティブ管理** | **編集・調整** | AIが生成したクリエイティブ（テキスト、画像）をブラウザ上で直感的に編集・調整できる。 |
| **高** | | **履歴・お気に入り** | 過去に生成したクリエイティブを一覧で確認し、検索やお気に入り登録ができる。 |
| **中** | **アセット管理** | **アセットライブラリ** | AIが生成時に参照するための参考資料（ロゴ、過去の優れた制作物、製品写真など）をアップロードし、一元管理する。 |

### 2.2. 画面構成案

| 画面 | URL | 概要 |
| :--- | :--- | :--- |
| **ブランドDNA設定** | `/brand-dna` | ブランドの理念やデザイン原則を入力・設定する、本システムの基盤となる画面。 |
| **クリエイティブ生成** | `/creatives/new` | 生成したいコンテンツの種類（例：SNS投稿）と目的・キーワードを入力し、AI生成を実行するメイン画面。 |
| **クリエイティブ一覧** | `/creatives` | 過去に生成したクリエイティブが一覧表示され、編集や再利用ができる画面。 |
| **アセット管理** | `/assets` | ロゴや参考画像をアップロード・管理する画面。 |

## 3. 非機能要件

| 項目 | 要件 |
| :--- | :--- |
| **UI/UX** | ・ユーザーがAIの存在を意識せず、自然な対話を通じてクリエイティブを生成できる体験を提供する。<br>・生成結果は複数パターン提示され、ユーザーが「選ぶ」「組み合わせる」ことで、より良いアイデアにたどり着けるよう支援する。 |
| **パフォーマンス** | ・テキスト生成は5秒以内、画像生成は15秒以内に初回結果を表示する。<br>・生成処理は非同期で行い、ユーザーの操作をブロックしない。 |
| **安定性** | ・AIの出力が不安定（フォーマット崩れ、品質のばらつき）な場合でも、アプリケーションとして破綻しない設計とする。<br>・生成結果が不適切な場合は再生成を促すなど、フォールバック機構を備える。 |
| **拡張性** | ・将来的に新しいクリエイティブ種別（動画シナリオ、プレゼン資料など）や、異なるAIモデルを追加・切り替え可能なアーキテクチャとする。 |
