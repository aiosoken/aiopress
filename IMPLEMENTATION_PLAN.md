# AIOプレス 実装計画書 (クリエイティブ生成特化)

## 1. はじめに

本ドキュメントは、要件定義書v3[1]およびシステム設計書v3[2]に基づき、AIOプレス（クリエイティブ生成特化版）を開発するための具体的な実装手順とマイルストーンを定義します。特に、短期間での開発が求められるハッカソンを想定した、現実的なステップを提示します。

## 2. 開発マイルストーン

開発を以下の3つのフェーズに分割して進めます。

- **フェーズ1: 環境構築と基盤開発 (1〜2日)**
- **フェーズ2: MVP（Minimum Viable Product）実装 (2〜3日)**
- **フェーズ3: 機能拡張と仕上げ (2日〜)**

--- 

## 3. フェーズ1: 環境構築と基盤開発

**目標**: 開発を開始できる状態を整え、アプリケーションの骨格を構築する。

| ステップ | タスク | 詳細 |
| :--- | :--- | :--- |
| **1.1** | **Google Cloud プロジェクト設定** | ・Google Cloud Consoleで新規プロジェクトを作成。<br>・課金を有効化。<br>・必要なAPI（IAM, Cloud Run, Cloud Functions, Vertex AI, Cloud Storage）を有効化。 |
| **1.2** | **Firebase プロジェクト設定** | ・Firebase Consoleでプロジェクトを作成し、Google Cloudプロジェクトに接続。<br>・Authentication（メール/パスワード）、Firestore、Storageを有効化。 |
| **1.3** | **Next.js プロジェクト初期化** | ・`npx create-next-app@latest` でプロジェクトを作成。<br>・TypeScript, Tailwind CSS, App Router を選択。<br>・`shadcn/ui` をセットアップし、基本的なUIコンポーネント（Button, Input, Card等）を導入。 |
| **1.4** | **Firebase SDK 導入** | ・`firebase` パッケージをインストール。<br>・Firebaseクライアントの初期化処理を実装 (`/lib/firebase/client.ts`)。 |
| **1.5** | **画面レイアウト構築** | ・`app/(main)/layout.tsx` に、サイドバーとヘッダーを含む基本的なダッシュボードレイアウトを構築。<br>・各画面 (`/brand-dna`, `/creatives/new` 等) の空のページファイルを作成。 |

--- 

## 4. フェーズ2: MVP実装

**目標**: コア機能である「ブランドDNAに基づくテキスト生成」を、最小限の構成で動作させる。

| ステップ | タスク | 詳細 |
| :--- | :--- | :--- |
| **2.1** | **ブランドDNA入力画面の実装** | ・`/brand-dna` ページに、`react-hook-form` と `zod` を用いて入力フォームを実装。<br>・入力項目: ミッション、トーン＆マナー、プライマリカラー等。<br>・「保存」ボタンクリックで、Firestoreの `/brands/{brandId}/dna/singleton` にデータを保存する処理を実装。 |
| **2.2** | **Cloud Function (生成API) の雛形作成** | ・`functions/src/generate.ts` に、HTTP `onCall` 関数を作成。<br>・フロントエンドから `brandId` とユーザー指示を受け取り、ログに出力するだけのダミー実装を行う。 |
| **2.3** | **クリエイティブ生成画面の実装** | ・`/creatives/new` ページに、生成指示（目的、キーワード等）を入力するフォームを実装。<br>・「生成」ボタンクリックで、上記Cloud Functionを呼び出す処理を実装。<br>・ローディング状態の管理と、ダミーのレスポンスを表示するUIを構築。 |
| **2.4** | **Gemini API 連携** | ・Cloud Function内で、Vertex AI SDK (`@google-cloud/vertexai`) を用いてGemini APIを呼び出す処理を実装。<br>・`lib/prompts/` にプロンプトテンプレートを作成し、ブランドDNAとユーザー指示を埋め込む。<br>・**最重要**: まずは「キャッチコピーを3つ生成する」機能に絞って実装する。 |
| **2.5** | **生成結果の表示と保存** | ・Gemini APIからのレスポンス（3つのキャッチコピー）をフロントエンドに返し、カード形式で表示する。<br>・生成された結果をFirestoreの `/brands/{brandId}/creatives/` コレクションに保存する。 |
| **2.6** | **クリエイティブ一覧画面の実装** | ・`/creatives` (トップページ) で、Firestoreから生成履歴を読み込み、一覧表示する。 |

**このフェーズ完了時点で、「ブランドDNAに基づいたキャッチコピー生成」というコアバリューのデモが可能になる。**

--- 

## 5. フェーズ3: 機能拡張と仕上げ

**目標**: MVPの機能を拡張し、より完全なプロダクトに近づける。

| ステップ | タスク | 詳細 |
| :--- | :--- | :--- |
| **3.1** | **SNS投稿文・ブログ記事生成** | ・Cloud Functionの `generate.ts` を拡張し、異なる生成種別に対応。<br>・`lib/prompts/` に、SNS投稿用、ブログ記事用のプロンプトテンプレートを追加。 |
| **3.2** | **Imagen API 連携 (画像生成)** | ・Cloud FunctionにImagen APIを呼び出す処理を追加。<br>・プロンプトにブランドカラーやスタイル指示を詳細に含める工夫を行う。<br>・生成された画像はCloud Storageに保存し、URLをFirestoreに記録する。 |
| **3.3** | **アセット管理機能の実装** | ・`/assets` ページで、ロゴなどの画像をCloud Storageにアップロードする機能を実装。<br>・生成時にこれらのアセットをAIが参照できるように、プロンプトに画像URLや説明を含める。 |
| **3.4** | **編集機能の強化** | ・生成されたテキストを編集するために、`Tiptap` などのリッチテキストエディタを導入。<br>・編集内容をFirestoreに保存する更新処理を実装。 |
| **3.5** | **UI/UXの改善** | ・`shadcn/ui` のコンポーネントを駆使し、全体的なデザインを洗練させる。<br>・ローディングスケルトンや、成功・エラー時のトースト通知などを実装。 |
