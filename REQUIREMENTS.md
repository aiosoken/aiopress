# AIOプレス 要件定義書 v2

## 1. 概要

### 1.1. プロジェクトの目的

本プロジェクトは、既存のWebアプリケーション「AIOプレス」を、提供された提案資料[1]および追加の仕様資料[3]に基づき、「**Brand Ops（ブランドオペレーション）基盤**」として全面的に再構築することを目的とする。新しい「AIOプレス」は、ブランドのトーン＆マナーを言語化・構造化し、AIが制作物を自動で評価・修正提案する仕組みを構築することで、クリエイティブ制作の効率化、ブランドイメージの統一、そしてAI時代における情報発信の最適化を実現する戦略的広報ソリューションを目指す。

### 1.2. 背景と課題

企業のブランド活動においては、部署や担当者、外部パートナーによって制作されるクリエイティブの品質やトーン＆マナーにばらつきが生じやすい。これにより、ブランドの一貫性が損なわれ、顧客に与える印象が希薄になるだけでなく、手戻りによる制作コストの増大という課題も存在する。本プロジェクトは、これらの課題をAIとデザインシステムの力で解決するものである。

### 1.3. スコープ

- **IN**:
    - 提案資料および追加仕様資料に基づく要件定義、設計、実装、テスト。
    - デザインシステム（デザイントークン）、ブランドガイドラインの管理機能。
    - AIによるクリエイティブの自動評価（スコアリング）および修正提案機能。
    - ブランドアセット（ロゴ、テンプレート等）の一元管理機能。
    - ダッシュボード、ユーザー管理、各種設定機能。
- **OUT**:
    - Epsonハードウェアとの連携機能（将来的な拡張機能として位置づける）。
    - 既存の `aiopress` レポジトリ[2]に含まれる、上記スコープ外の機能。

## 2. 機能要件

追加資料[3]で提示された画面構成に基づき、以下の機能要件を定義する。

| 画面 | 機能名 | 機能概要 | 優先度 |
| :--- | :--- | :--- | :--- |
| **ダッシュボード (`/`)** | **統計サマリー** | 総チェック数、平均ブランド準拠スコア、ガイドライン数、デザイントークン数などの主要KPIをカード形式で表示する。 | 高 |
| | **最近のチェック結果** | 直近に行われたAIチェックの結果（ファイル名、スコア、ステータス）を一覧で表示する。 | 高 |
| | **クイックアクション** | 「新規チェック」「ガイドライン編集」など、主要機能へのショートカットボタンを配置する。 | 高 |
| **ガイドライン (`/guidelines`)** | **ガイドライン管理** | ブランドのルール（アイデンティティ、色彩、タイポグラフィ、ロゴ規定など）をセクションごとに作成・編集・バージョン管理できる。 | 高 |
| **トークン管理 (`/tokens`)** | **デザイントークン管理** | デザインシステムの中核となるデザイントークン（色、タイポグラフィ、余白など）をキーと値のペアで管理する。 | 高 |
| | **トークンエクスポート** | 登録されたデザイントークンをJSON形式でエクスポートし、Figmaや開発環境で利用できるようにする。 | 中 |
| **アセット管理 (`/assets`)** | **アセットライブラリ** | ロゴ、アイコン、テンプレート、写真などのブランドアセットをアップロードし、タグ付けや検索ができるように一元管理する。 | 高 |
| **AIチェック (`/check`)** | **クリエイティブ評価** | 制作したクリエイティブ（画像、PDF）をアップロードし、AIがガイドラインやデザイントークンへの準拠度を自動で評価し、スコアを算出する。 | 高 |
| | **修正提案** | 評価結果に基づき、具体的な問題点（例：「使用されている色がプライマリカラーと異なります」）と修正案を提示する。 | 高 |
| **設定 (`/settings`)** | **プロジェクト設定** | プロジェクト名や組織名などの基本情報を設定する。 | 中 |
| | **チェック設定** | AIチェックの合格スコアの閾値や、自動チェックのON/OFFなどを設定できる。 | 中 |
| | **ユーザー管理** | 複数の教職員が共同で利用できるよう、チームメンバーの招待および権限（管理者、編集者）の管理ができる。 | 中 |

## 3. 非機能要件

| 項目 | 要件 | 測定指標 |
| :--- | :--- | :--- |
| **UI/UX** | ブランド担当者がマニュアルなしで直感的に操作できる、モダンでクリーンなインターフェースを提供する。 | ユーザーテストにおけるタスク完了率95%以上。 |
| **パフォーマンス** | AIによるチェック処理は非同期で行い、ユーザーを待たせない。チェック結果はリアルタイムに画面に反映される。 | AIチェックのリクエストから結果表示まで10秒以内。 |
| **セキュリティ** | ブランド資産、個人情報、AIモデルへのアクセスキーなどの機密情報を安全に管理する。Firebaseのセキュリティルールを適切に設定し、不正アクセスを防止する。 | 定期的な脆弱性診断の実施。 |
| **デザインの一貫性** | アプリケーション全体で、設定されたデザインシステムに基づいた一貫性のあるビジュアルを提供する。 | `shadcn/ui` をベースにしたカスタムコンポーネントライブラリを構築し、再利用を徹底する。 |
| **拡張性** | 将来的に新しいチェック項目や外部サービスを追加しやすいよう、モジュール化されたアーキテクチャを維持する。 | 新機能追加時の平均開発工数。 |
